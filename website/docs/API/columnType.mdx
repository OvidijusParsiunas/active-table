# Column Type

Properties related to column types:

### `defaultColumnTypeName` {#defaultColumnTypeName}

- Type: `string`
- Default: _First available type to not have validation -> First available type -> "Text"_

Nname for the default column type.

### `availableDefaultColumnTypes` {#availableDefaultColumnTypes}

- Type: [`DEFAULT_COLUMN_TYPES[]`](#DEFAULT_COLUMN_TYPES)
- Default: _["Text", "Number", "Currency", "Date d-m-y", "Date m-d-y", "Checkbox", "Select", "Label"]_

Used to overwrite the available default column types. If array is set to empty (_[]_) and there are no [`customColumnTypes`](#customColumnTypes), the default type
will be automatically set to _"Text"_.

#### Example

<TableContainer>
  <active-table
    availableDefaultColumnTypes='["Number"]'
    content='[
      ["Planet", "Diameter", "Mass", "Moons", "Density"],
      ["1", 12756, 5.97, 1, 5514],
      ["2", 6792, 0.642, 2, 3934],

      ["5", 49528, 102, 14, 1638]]'
    tableStyle='{"width":"100%", "boxShadow": "rgb(172 172 172 / 17%) 0px 0.5px 1px 0px", "borderRadius":"2px"}'
  ></active-table>
</TableContainer>

### `customColumnTypes` {#customColumnTypes}

- Type: [`ColumnType[]`](#ColumnType)
- Default: _[]_

An array of custom column types which are automatically appended to the [`availableDefaultColumnTypes`](#availableDefaultColumnTypes) types array.

## Types

Shared object property types related to column types:

### `ColumnType` {#ColumnType}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `name: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`textValidation?: TextValidation`](#TextValidation), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`customTextProcessing?: CustomTextProcessing`](#CustomTextProcessing), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`sorting?: Sorting`](#Sorting), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`iconSettings?: IconSettings`](#IconSettings), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`select?: boolean|CellDropdown<SelectOptions>`](#CellDropdown), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`label?: boolean|CellDropdown<LabelOptions>`](#CellDropdown), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`calendar?: Calendar`](#Calendar), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`checkbox?: Checkbox`](#Checkbox) <br />
  }

- Default: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; _sorting: in the ASCII character order ("Text" type)_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _dropdownIconSettings: using "Text" type icon _<br />
  }

Details for creating a column type. <br />
If [`textValidation`](#TextValidation) and [`customTextProcessing`](#CustomTextProcessing) are both provided, [`customTextProcessing`](#CustomTextProcessing) will be executed first. <br />
The [`select`](#CellDropdown), [`label`](#CellDropdown), [`calendar`](#Calendar) and [`checkbox`](#Checkbox) properties facilitate unique features and if
more than one of these is provided - only the first one will be used in that order.

<LineBreak></LineBreak>

### `TextValidation` {#TextValidation}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `func: (cellText: string, rowIndex: number) => boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `setTextToDefaultOnFail?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`failedStyle?: NoDimensionCSSStyle`](./sharedTypes#NoDimensionCSSStyle), <br />
  }
- Default: { _setTextToDefaultOnFail: true_ }

Used to enforce cell text validation. This operates only in the data cells and not the header. <br />
The `func` property takes a function that will be called when the text is initially placed in the cell and when
it changes. The function must return _true_ if the text is correct or _false_ if it is incorrect.
`setTextToDefaultOnFail` will reset the cell text to value defined inside [`defaultText`](./content#defaultText)
if the function returns _false_. <br />
`failedStyle` is used to change the cell styling if the function returns _false_.

#### Example validation for if number is higher than 20

<TableContainer>

    customColumnTypes='[

    defaultColumnTypeName="Custom type"
    content='[

### `CustomTextProcessing` {#CustomTextProcessing}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `changeTextFunc?: (cellText: string, rowIndex: number) => string|number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`changeStyleFunc?: (cellText: string, rowIndex: number) => NoDimensionCSSStyle`](./sharedTypes#NoDimensionCSSStyle), <br />
  }

Augments cell after its text is changed. `changeTextFunc` is a function that processes cell text and
returns same/new text depending on its contents. `changeStyleFunc` is a function that processes cell text and
returns a styling object for the cell depenging on its contents.

#### Example processing for currency: 20 (red) < 100 (yellow) < more (green)

<TableContainer>
  <active-table
    customColumnTypes='[
      {
        "name": "Custom type",
        "customTextProcessing": {
          "changeTextFunc": "(cellText) => `$${Number(cellText.replace(/[^0-9.-]+/g,\"\"))}`",
          "changeStyleFunc": "(cellText) => {
            const moneyNumber = Number(cellText.replace(/[^0-9.-]+/g,\"\"));
            if (moneyNumber < 20) return {backgroundColor: `#ff3232`};
            if (moneyNumber < 100) return {backgroundColor: `#ffff00`};
            return {backgroundColor: `#00ac00`};
          }"
      }}]'
    defaultColumnTypeName="Custom type"
    content='[
      ["Gravity", "Diameter", "Mass", "Moons", "Density"],
      [9.8, 12756, 5.97, 1, 5514],
      [3.7, 6792, 0.642, 2, 3934],
      [23.1, 142984, 1898, 79, 1326],
      [9, 120536, 568, 82, 687],
      [11, 49528, 102, 14, 1638]]'
    tableStyle='{"width":"100%", "boxShadow": "rgb(172 172 172 / 17%) 0px 0.5px 1px 0px", "borderRadius":"2px"}'

### `Sorting` {#Sorting}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `ascendingFunc: (cellText1: string, cellText2: string) => number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `descendingFunc: (cellText1: string, cellText2: string) => number`, <br />
  }

Facilitates column cell sorting capabilities.
The functions are executed using the standard JavaScript [.sort](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/sort)
method and are expected to return a negative value (< 0) if cellText1 is less than cellText2, zero (0) if both arguments
are equal, or a positive value (> 0) if cellText1 is greater than cellText2.

#### Example sorting for numbers

### `IconSettings` {#IconSettings}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`reusableIconName?: DEFAULT_COLUMN_TYPES`](#DEFAULT_COLUMN_TYPES), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `svgString?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `containerStyles?`: { <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [`columnDropdown?: CSSStyle`](./sharedTypes#CSSStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [`headerCorrections?: CSSStyle`](./sharedTypes#CSSStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp;}}

Defines column type icon properties. <br />
`reusableIconName` allows the icon to reuse of the existing default type icons. <br />
`svgString` is the full svg element string containing all DOM tags. Make sure to escape the quotes - change " to \" (An easy way to do this is to
stringify it using JSON.stringify()). If the string is defined directly in HTML, you will need to escape the first less than (<) character by replacing it with `&lt`. <br />
`containerStyles` allows the icon dimensions to be customised when displayed in the `columnDropdown` and in the header. By default, the `columnDropdown` dimension
properties are also used for the header however if they don't fit the `headerCorrections` object can be used to change them.

### `CellDropdown` {#CellDropdown}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`dropdownStyle?: CellDropdownStyle`](#CellDropdownStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `optionStyle?: {textColor: string}`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`options?: SelectOptions|LabelOptions`](#SelectOptions), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `canAddMoreOptions?: boolean` <br />
  }
- Default: { _canAddMoreOptions: true or if options property is set then false_ }

Dropdown customization properties for `select` and `label` properties in the [`ColumnType`](#ColumnType) object. <br />
`dropdownStyle` and `optionStyle` are used to apply specific dropdown styling aspects. <br />
`options` property defines custom options within the dropdown and uses the [`SelectOptions`](#SelectOptions) type to define individual options when
used for a `select` dropdown and [`LabelOptions`](#LabelOptions) if used for `label`. <br />
`canAddMoreOptions` controls whether new options can be added into the dropdown. If _false_ then column cell text that
is not in the _options_ array is automatically removed.

#### `CellDropdownStyle` {#CellDropdownStyle}

- Type: { <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `width?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `paddingTop?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `paddingBottom?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `marginTop?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `marginLeft?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `textAlign?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `border?: string`, <br />
  }

Styling properties for the select and label dropdowns.

#### `SelectOptions` {#SelectOptions}

- Type: `string[]`<br />

### `Calendar` {#Calendar}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `toYMDFunc: (cellText: string) => YMDFormat`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `fromYMDFunc: (YMD: YMDFormat) => string`, <br />
  }

Adds a hoverable calendar icon element to each data cell. When a cell is focused - a [date picker overlay](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/date)
is opened which the user can use to help select the date they need. <br />
The `toYMDFunc` and `fromYMDFunc` functions are strictly used for converting cell text into a format ([`YMDFormat`](#YMDFormat)) the calendar API
accepts and back to the original format when a new date is chosen.

#### `YMDFormat` {#YMDFormat}

- Type: `[string, string, string, string?, string?, string?, string?]`

Date format used by the [`Date Picker`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/date). The tuple contents translate to
year / month / day (as digits e.g. '2022') and optionally hours / minutes / seconds / milliseconds (not used by the picker but may be required
for a custom [`Calendar`](#Calendar) type).

#### Example for custom d/m/y type

<TableContainer>

### `Checkbox` {#Checkbox}

- Type: `boolean`

Replaces cell text with a checkbox. <br />
This type automatically comes with a predefined [`customTextProcessing`](#CustomTextProcessing) property in the [`ColumnType`](#ColumnType) that
uses _""_, _"0"_, _"00"_ and _"false"_ values to represent the checkbox as unchecked and all the other values as checked. You can overwite it
with your own `changeTextFunc` function that would return a string of _"true"_ when the checkbox should be checked and _"false"_ when it should
be unchecked. <br />
The [`onCellUpdate`](./events#onCellUpdate) and [`onContentUpdate`](./events#onContentUpdate) properties will return string _"true"_ if the checkbox
is checked and _"false"_ if unchecked.
