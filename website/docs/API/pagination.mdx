# Pagination

Divide table rows into a series of pages.

### `pagination` {#pagination}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp;`rowsPerPage?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;[`rowsPerPageSelect?: boolean|RowsPerPageSelect`](#RowsPerPageSelect), <br />
  &nbsp;&nbsp;&nbsp;&nbsp;`maxNumberOfVisiblePageButtons?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;`displayPrevNext?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;`displayFirstLast?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;`displayNumberOfVisibleRows?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;[`style?: PaginationStyle`](#PaginationStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp;`positions?: PaginationPositions` <br />
  }
- Default: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp;_rowsPerPage: 10_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;_rowsPerPageSelect: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;_maxNumberOfVisiblePageButtons: 8_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;_displayPrevNext: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;_displayFirstLast: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;_displayNumberOfVisibleRows: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;_positions: refer to the PaginationPositions type_ <br />
  }

`rowsPerPage` is the number of maximum data rows that can be displayed in a single page. <br />
`rowsPerPageSelect` facilitates a `rowsPerPage` selection button that opens up a dropdown containing a number of options that the user can select
to change `rowsPerPage`. <br />
`maxNumberOfVisiblePageButtons` is the maximum number of page (number) buttons that are visible, the rest will be hidden. <br />
`displayPrevNext` defines whether the previous and next page buttons are displayed. <br />
`displayFirstLast` defines whether the first and last page buttons are displayed. <br />
`displayNumberOfVisibleRows` displays information on the index of currently visible rows out of the total number of rows in the table. E.g. _1-5 of 20_. <br />
`style` defines the styling properties of pagination components. <br />
`positions` defines pagination components positions. <br />

## Types

Object types for properties inside [`pagination`](#pagination):

### `RowsPerPageSelect` {#RowsPerPageSelect}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp;`prefixText?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;`options?: (number | "all" | "All" )[]` <br />
  }
- Default: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp;_prefixText: "Rows per page"_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;_options: [10, 25, 50, "All"]_ <br />
  }

Custom settings for the `rowsPerPageSelect` property. <br />
`prefixText` is the text that is displayed before the button. <br />
`options` is the array of options that are displayed in the dropdown. `"all"|"All"` option displays all of the table's rows within one page.
If the defined `rowsPerPage` value (in [`pagination`](#pagination) object) is not in the options array, it will be added to it automatically. <br />

### `PaginationStyle` {#PaginationStyle}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp;[`rowsPerPageSelect?: RowsPerPageOptionsStyle`](#RowsPerPageOptionsStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp;[`numberOfVisibleRows?: CSSStyle`](./types#CSSStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp;`pageButtons?: PageButtonStyle` <br />
  }

Styling properties of pagination components. <br />
`rowsPerPageSelect` defines styling properties for the rowsPerPage button container and its dropdown.
`numberOfVisibleRows` defines styling properties for the number of visible rows text. <br />
`pageButtons` defines styling properties for the page/previous/next/first/last buttons. <br />

:::tip
When number of visible rows text contains digits that are in the 100s+, it is advisable to set `numberOfVisibleRows` with a minWidth value that is higher than
'32px' to stop side components from moving when its digits change.
:::

:::tip
When setting button side borders and a container border - use borderRight as buttons have a default borderRight style which when using with
a borderLeft would cause an unexpectadly thick border - alternatively you can simply unset borderRight.
:::
