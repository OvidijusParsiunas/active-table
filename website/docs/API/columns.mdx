### `preserveNarrowColumns`

- Type: `boolean`
- Default: _true_

When [`tableStyle`](./table#tableStyle) is set with a `width` or `maxWidth` property to limit its size, the insertion of new columns will
cause all existing column widths to automatically be reduced to make space for the new ones (except columns set with `isResizable` as _false_
inside [`cellStyle`](#CellStyle) of the [`customColumnsSettings`](#customColumnsSettings) object). This can however become problematic when existing column
widths eventually become too narrow\* and their widths cannot be reduced any further. <br />
When `preserveNarrowColumns` is `true` and column widths are too narrow\*, the table width will be increased allow new columns to be inserted. <br />
When `preserveNarrowColumns` is `false` and column widths are too narow\*, no new columns will be allowed to be inserted.
set to true by default.<br />
_\*Minimal column size for most browsers is 33px_

**Example where preset table width of 300px is exceeded**

## Types

Object types that are shared by [`columnsSettings`](#columnsSettings) and [`customColumnsSettings`](#customColumnsSettings) properties:

### `ColumnSettings` {#ColumnSettings}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `defaultText?: string|number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `isDefaultTextRemovable?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `isCellTextEditable?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `isHeaderTextEditable?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`cellStyle?: CellStyle`](#CellStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `isResizable?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`headerStyles?: HoverableStyles`](#HoverableStyles), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`headerIconStyle?: HeaderIconStyle`](#HeaderIconStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`defaultColumnTypes?: DEFAULT_COLUMN_TYPES[]`](#DEFAULT_COLUMN_TYPES), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`customColumnTypes?: ColumnType[]`](#ColumnType), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `activeTypeName?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`dropdown?: ColumnDropdownSettingsDisplay`](#ColumnDropdownSettings) <br />
  }
- Default: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; _defaultText: ""_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _isDefaultTextRemovable: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _isCellTextEditable: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _isHeaderTextEditable: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _isResizable: true_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _defaultColumnTypes: ["Text", "Number", "Currency", "Date d-m-y", "Date m-d-y", "Checkbox", "Select", "Label"]_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _activeTypeName: First type to not have validation -> First available type -> "Text"_, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; _dropdown: {displaySettings: {isAvailable: true, openMethod: {cellClick: true}}_, <br />
  }

`defaultText` is text that is displayed when there is the cell has no text or if its type validation fails. <br />
`isDefaultTextRemovable` removes the default text when the cell is focused. <br />
`isCellTextEditable` defines if cell text can be edited and applies to both data and header cells, but is superseded by
`isHeaderTextEditable` for header cells. <br />
`cellStyle` defines the cell style and applies to both data and header cells, but is superseded by `headerStyles` for
header cells which defines styling for default and mouse hover states. <br />
`headerIconStyle` is used to overwrite the style of the header icon if present ([`displayIconsInHeaders`](./header#displayIconsInHeaders)). <br />
`isResizable` determines if the column can be resized by using the overlay between header cells. <br />
`defaultColumnTypes` overwrites the available default column types (_use [] to remove all_) and `customColumnTypes` defines additional custom
column types. <br />
`activeTypeName` is the name for the initial on-render type of the column. <br />

#### Example

      "defaultColumnTypes": ["Text"]

### `CellStyle` {#CellStyle}

- Type: { `CSSStyle` & `!minWidth` & `!maxWidth` & `!height` & `!minHeight` & `!maxHeight` }

Object containing all `CSSStyle` properties except `minWidth`, `maxWidth`, `height`, `minHeight` and `maxHeight`. <br />
This object also serves as the default style for auxiliary styles (except for `width`) WORK. <br />
The `width` property supports _px_ and _%_ metrics, however when using _%_ - the [`tableStyle`](./table#tableStyle) object should have its `width` property
also set as otherwise the cells will not be able to calculate a relative width. Additionally, when [`CellStyle`](#CellStyle) - `width` and
[`tableStyle`](./table#tableStyle) - `width` are defined and `isResizable` in [`ColumnSettings`](#ColumnSettings) is set to _true_, the [`CellStyle`](#CellStyle) - `width`
will be overriden to reach the set table `width`, hence to prevent this make sure to set `isResizable` to _false_.

#### Example

<TableContainer>
  <active-table
    columnsSettings='{
      "cellStyle": { "borderRight": "1px solid grey", "width": "110px" },
      "isResizable": false
    }'
    customColumnsSettings='[
      { "headerName": "Moons", "isResizable": true },
      { "headerName": "Density", "isResizable": true }
    ]'
    content='[
      ["Planet", "Diameter", "Mass", "Moons", "Density"],

      ["Jupiter", 142984, 1898, 79, 1326],

  &nbsp;&nbsp;&nbsp;&nbsp; _dropdownIconSettings: using "Text" type icon _<br />
  }

Details for creating a column type. <br />
If `textValidation` and `customTextProcessing` are both provided, `customTextProcessing` will be executed first. <br />
The `select`, `label`, `calendar` and `checkbox` properties facilitate unique features and if
more than one of these is provided - only the first one will be used in that order.

### `TextValidation` {#TextValidation}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `func: ((cellText: string) => boolean) as string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `setTextToDefaultOnFail?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`failedStyle?: NoDimensionCSSStyle`](./types#NoDimensionCSSStyle), <br />
  }
- Default: { _setTextToDefaultOnFail: true_ }

Used to enforce cell text validation. This operates only in the data cells and not the header. <br />
The `func` property takes a function that will be called when the text is initially placed in the cell and when
it changes. The function must return `true` if the text is correct or `false` if it is incorrect. It is IMPORTANT
to note that the function needs to be converted into a string, as otherwise it will be removed when passed in
with the parent object.
`setTextToDefaultOnFail` will reset the cell text to value defined inside [`ColumnSettings` -> `defaultText`](#ColumnSettings)
if the function returns `false`. <br />
`failedStyle` is used to change the cell styling if the function returns `false`.

### `CustomTextProcessing` {#CustomTextProcessing}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `changeTextFunc?: ((cellText: string) => string|number) as string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`changeStyleFunc?: ((cellText: string) => NoDimensionCSSStyle) as string`](./types#NoDimensionCSSStyle), <br />

### `Sorting` {#Sorting}

- Type: {<br />

  &nbsp;&nbsp;&nbsp;&nbsp; `descendingFunc: ((cellText1: string, cellText2: string) => number) as string`, <br />

are equal, or a positive value (> 0) if cellText1 is greater than cellText2.

### `CellDropdown` {#CellDropdown}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`dropdownStyle?: CellDropdownStyle`](#CellDropdownStyle), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `optionStyle?: {textColor: string}`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`options?: SelectOptions|LabelOptions`](#SelectOptions), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `canAddMoreOptions?: boolean` <br />
  }
- Default: { _canAddMoreOptions: true or if options property is set then false_ }

Dropdown customization properties for `select` and `label` properties in the [`ColumnType`](#ColumnType) object. <br />
`dropdownStyle` and `optionStyle` are used to apply specific dropdown styling aspects. <br />
`options` property defines custom options within the dropdown and uses the [`SelectOptions`](#SelectOptions) type to define individual options when
used for a `select` dropdown and [`LabelOptions`](#LabelOptions) if used for `label`. <br />
`canAddMoreOptions` controls whether new options can be added into the dropdown. If _false_ then column cell text that
is not in the _options_ array is automatically removed.

#### `CellDropdownStyle` {#CellDropdownStyle}

- Type: { <br />

### `Calendar` {#Calendar}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `toYMDFunc: ((cellText: string) => YMDFormat) as string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `fromYMDFunc: ((YMD: YMDFormat) => string) as string`, <br />
  }

Adds a hoverable calendar icon element to each data cell. When a cell is focused - a [date picker overlay](#https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/date)
is opened which the user can use to help select the date they need. <br />
The `toYMDFunc` and `fromYMDFunc` functions are strictly used for converting cell text into a format ([`YMDFormat`](#YMDFormat)) the calendar API
accepts and back to the original format when a new date is chosen.
It is IMPORTANT to note that both functions need to be converted into strings as otherwise they will be removed when passed in with the parent object.

#### `YMDFormat` {#YMDFormat}

- Type: `[string, string, string, string?, string?, string?, string?]`

Date format used by the [`Date Picker`](#https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/date). The tuple contents translate to
year / month / day (as digits e.g. '2022') and optionally hours / minutes / seconds / milliseconds (not used by the picker but may be required
for a custom [`Calendar`](#Calendar) type).

#### Example for custom d/m/y type

<TableContainer>
  <active-table
    columnsSettings='{

### `Checkbox` {#Checkbox}

- Type: `boolean`

Replaces cell text with a checkbox. <br />
This type automatically comes with a predefined [`customTextProcessing`](#CustomTextProcessing) property in the [`ColumnType`](#ColumnType) that
uses _""_, _"0"_, _"00"_ and _"false"_ values to represent the checkbox as unchecked and all the other values as checked. You can overwite it
with your own `changeTextFunc` function that would return a string of "true" when the checkbox should be checked and "false" when it should
be unchecked. <br />
The `onCellUpdate` WORK property will return string "true" if the checkbox is checked and "false" if unchecked.
